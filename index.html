Project Summary — Independently designed, built, and deployed a complete Progressive Web App for a real Hong Kong beauty salon. The system handles clients, treatments, appointment booking with 3D collision detection, installment payment tracking, refunds, revenue reports, and staff scheduling. Built with React 18 + Vite on the frontend, Supabase (PostgreSQL) on the backend, and deployed via Vercel CI/CD. All at zero monthly cost using free cloud tiers.

Key Features
Designed around the real daily workflow of a Hong Kong beauty salon.

3D Collision-Check Booking

Real-time conflict detection across staff, room, and equipment dimensions. Visual indicators on DayPicker calendar.

Installment Payment Tracking

Free-input payment amounts with progress bar (paid / total). Multi-method: Cash, Card, Transfer, PayMe, AlipayHK.

INSERT-ONLY Activity Log

Tamper-proof audit trail — RLS prohibits UPDATE/DELETE on log records. All money and session changes atomically recorded.

Refund Workflow

Free-input refund amount with automatic session restoration, mandatory reason, and payment method. Atomically logged.

WhatsApp Snippet Selector

Smart tag chips with {{variable}} injection, inline CRUD, and one-tap copy for customer communication.

Excel Export & Dormant Detection

One-click XLSX reports. Auto-detection of clients inactive for 90+ days with WhatsApp quick-action button.

3-Layer Treatment Deactivation

Block if in use (show who), soft-disable if historical, allow delete only if unused. Prevents data corruption.

Touch-Friendly Discount Calculator

Slider-based % OFF with live 3-way sync (unit price, discount, package price). Built for iPad with 48px touch targets.

Security Architecture
6-layer defense-in-depth, from browser input validation to database constraints.

Layer 1: Authentication

Supabase JWT with Magic Link + PASSWORD_RECOVERY flow. Session auto-refresh with onAuthStateChange guard.

Layer 2: Authorization

Row-Level Security on all 11 tables. Role-based policies (shop_owner / staff). Custom Claims in JWT app_metadata.

Layer 3: Data Integrity

CHECK constraints (price > 0, capacity > 0, duration > 0). NOT NULL on critical fields. Frontend + DB dual input validation.

Layer 4: Audit Trail

activity_log table — INSERT-ONLY. RLS denies UPDATE / DELETE on audit records. All sensitive mutations permanently logged.

Layer 5: Atomic Operations

15+ SECURITY DEFINER RPC functions. FOR UPDATE row locks prevent race conditions. Idempotent consumption checks.

Layer 6: Key Management

ANON key (browser-safe, RLS-protected). SECRET key (server-only, never exposed). Git history cleaned, .env in .gitignore.
