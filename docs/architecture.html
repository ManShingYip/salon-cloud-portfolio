3 Multi-Layer Security Architecture

🛡️ Layer 1: Authentication
  • Supabase JWT (JSON Web Token)
  • Magic Link + Password Reset flow
  • Session auto-refresh
🔑 Layer 2: Authorization
  • Row-Level Security (RLS) — all 11 tables
  • Role-based policies (shop_owner / staff)
  • Custom Claims in JWT app_metadata
🔒 Layer 3: Data Integrity
  • CHECK constraints (price > 0, capacity > 0)
  • NOT NULL constraints on critical fields
  • Frontend + backend dual input validation
📋 Layer 4: Audit Trail
  • activity_log — INSERT-ONLY
  • RLS denies UPDATE / DELETE
  • All money/session mutations logged
⚛️ Layer 5: Atomic Operations
  • SECURITY DEFINER RPC functions
  • FOR UPDATE row locks
  • Idempotent consumption checks
🔐 Layer 6: Key Management
  • ANON key → browser-safe, RLS-protected
  • SECRET key → server-only, never exposed
  • Git history cleaned · .env in .gitignore
